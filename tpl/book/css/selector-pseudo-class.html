语法

伪类选择器语法主要形式为：

E.class:pseudo-class {
    property: value;
}

即：

选择器:伪类 {
    样式属性: 样式值;
}

下面开始我们一起来看看这些伪类的具体应用：

动态伪类

动态伪类只有当用户和网站交互的时候才能体现出来。

动态伪类包含两种：

  • 锚点伪类，如 :link、:visited
  • 行为伪类，如 :hover、:active 和 :focus。

锚点伪类

/* 链接没有被访问时前景色为灰色 */
.demo a:link {
    color: gray;
}

/* 链接被访问过后前景色为黄色 */
.demo a:visited {
    color: yellow;
}

行为伪类


/* 焦点在目标上时前景色为红色 */
.demo a:focus {
    color: red;
}

/* 鼠标悬浮在目标上时前景色为绿色 */
.demo a:hover {
    color: green;
}

/* 鼠标点击目标时的前景色为蓝色 */
.demo a:active {
    color: blue;
}

注意点

定义的先后顺序需要符合 link-visited-hover-active，因为定义在后面的伪类的样式将覆盖之前的状态，而这四种状态每个后者都包含了前者的状态。

:focus 和 :hover 的顺序，则需要根据具体应用，焦点和鼠标悬浮状态同时出现时，需要展现哪种样式，哪种样式就写在后面。

触发 :focus 的条件

w3c 标准中，能够触发 focus 事件的元素或对象有：

  1. 包含 href 属性的 HTMLAnchorElement/HTMLAreaElement
  2. 不包含 disabled 属性的 HTMLInputElement/HTMLSelectElement/HTMLTextAreaElement/HTMLButtonElement
  3. HTMLIFrameElement
  4. 所有包含 tabindex 属性但属性值不能为 -1 的元素
  5. window 对象 但是各个浏览器支持情况不尽相同，浏览器支持情况： http://www.quirksmode.org/dom/events/blurfocus.html

:hover 在 IE6 下只有 a 元素支持，:active、:focus IE6 与 IE7 不支持。

UI 元素状态伪类

我们把 :enabled、:disabled、:checked 伪类称为 UI 元素状态伪类。

这些主要是针对于 HTML 中的表单元素，可用、不可态状态。

另外单选框和多选框有选中和非选中两种状态。

来看两个实例，比如说你想将 disabled 的文本框与别的文本框区别出来，你就可以这样应用

input:disabled {
    border:1px solid #999;
    background-color: #fefefe;
}

这样一来就把页面中禁用的文本框就应用了一个不同的样式。

IE6-8 不支持 :enabled、:disabled、:checked 这三种选择器。

:nth 选择器

这部分内容是 CSS3 加入的新选择器，有人也称这种选择器为 CSS3 结构类，下面我们通过实际的应用来具体了解他们的使用和区别，首先列出他具有的选择方法：

  • :first-child ：选择某个元素集合的第一个子元素；
  • :last-child ：选择某个元素集合的最后一个子元素；
  • :nth-child() ：选择某个元素集合的一个或多个特定的子元素；
  • :nth-last-child() ：选择某个元素集合的一个或多个特定的子元素，从这个元素的最后一个子元素开始算；
  • :nth-of-type() ：选择指定的元素；
  • :nth-last-of-type() ：选择指定的元素，从元素的最后一个开始计算；
  • :first-of-type ：选择一个上级元素下的第一个同类子元素；
  • :last-of-type ：选择一个上级元素的最后一个同类子元素；
  • :only-child ：选择的元素是它的父元素的唯一一个了元素；
  • :only-of-type ：选择一个元素是它的上级元素的唯一一个相同类型的子元素；
  • :empty ：选择的元素里面没有任何内容。

:first-child

:first-child 是用来选择某个元素集合的第一个子元素。

在我们样例中，你想让列表中的"1"具有与从不同的样式：

.demo li:first-child {
    background: green;
    border: 1px dotted blue;
}

请看效果：

IE6 不支持 :first-child 选择器。

:last-child

:last-child 选择器与 :first-child 选择器的作用类似，不同的是选择是的元素集合的最后一个子元素。

在我们的样例中需要单独给列表最后一项一个不同的样式，我们就可以使用这个选择器，如：

.demo li:last-child {
    background: green;
    border: 2px dotted blue;
}

:nth-child()

:nth-child() 可以选择某个的一个或多个特定的子元素，你可以按以下方式进行选择：

我们定义 $i、$j 为某个整数

  • :nth-child($i) ：参数是具体数字
  • :nth-child(n) ：参数是 n，n 代表非负整数
  • :nth-child(n*$i) ：选择 $i 的倍数，n 代表非负整数
  • :nth-child(n+$i) ：选择大于 $i 后面的元素
  • :nth-child(-n+$i) ：选择小于 $i 前面的元素
  • :nth-child(n*$i+$j) ：表示隔 $i 选第 $j 个

:nth-child() 可以定义他的值（值可以是整数，也可以是表达式）。

:nth-child(3)，选择某元素下的第三个子元素，比如需要选择列表中的第三个 li 元素，那么我们可以直接这样使用：

.demo li:nth-child(3) {
    background: lime;
}

效果如下所示：

这种形式不能使用负值，也就是说 li:nth-child(-3) 是错误的。

:nth-child(n)，其中 n 是一个简单的表达式，那么 n 代表非负整数，此时将会选中所有子元素。

在我们的例子中，在 li 中使用 :nth-child(n)，那么将选中所有的 li，如：

.demo li:nth-child(n) {
    background: lime;
}

等于

.demo li {
    background: lime;
}

他其实是这样计算的

n = 0 -> 没有选择元素

n = 1 -> 选择第一个 li

n = 2 -> 选择第二个 li，以此类推，这样下来就选中了所有的 li

请看效果：

注意

  • 这里的 n 不能使用其他字母代替。

:nth-child(2n)，我们可以选择第 2n 倍数的元素，其中 2 可以换成你自己需要的正整数，如：

.demo li:nth-child(2n) {
    background: lime;
}

等于

.demo li:nth-child(even) {
    background: lime;
}

我们来看一下其计算的过程：

n = 0 -> 2n = 0 -> 没有选中任何元素

n = 1 -> 2n = 2 -> 选择了第二个 li

n = 2 -> 2n = 4 -> 选择了第四个 li，后面的依此类推

如图所示：

:nth-child(2n) 也等于 :nth-child(even) 效果。

:nth-child(2n-1),这个选择器是在":nth-child(2n)"基础上演变过来的，上面说了人是选择偶数，那么我们在他的基础上减去“1”就变成奇数选择，如：

.demo li:nth-child(2n-1) {
    background: lime;
}

我们来看看其实现过程

n = 0 -> 2n-1 = -1 -> 也没有选中任何元素，

n = 1 -> 2n-1 = 1 -> 选择第一个 li

n = 2 -> 2n-1 = 3 -> 选择第三个 li,后面的依此类推

其实实现这种奇数效果，我们还可以使用 :nth-child(2n+1) 或 :nth-child(odd)，一起来看他们的效果

:nth-child(n+5) 这个选择器是选择从第五个元素开始选择，这里的数字你可以自己定义，如：

.demo li:nth-child(n+5) {
    background: lime;
}

按前面的计算方法，我们来看看，

n = 0 -> n+5 = 5 -> 选中第5个 li

n = 1 -> n+5 = 6 -> 选择第6个 li，后面的就不列出来了，原理一样

你可以使用这种方法选择你需要开始选择的元素位置，也就是说换了数字，起始位置就变了，看下在的效果图：

:nth-child(-n+5) 这种选择器刚好和上面的选择器相反，这个是选择第5个前面的，如：

.demo li:nth-child(-n+5) {
    background: lime;
}

如果不清楚怎么一回事，你只要计算一下就明白了

n = 0 -> -n+5 = 5 -> 选择了第5个 li

n = 1 -> -n+5 = 4 -> 选择了第4个 li

n = 2 -> -n+5 = 3 -> 选择了第3个 li

n = 3 -> -n+5 = 2 -> 选择了第2个 li

n = 4 -> -n+5 = 1 -> 选择了第1个 li

n = 5 -> -n+5 = 0 -> 没有选择任何元素

从上面的计算方法中，大家很清楚的知道是怎么得来的，最后我们一起看看效果吧：

:nth-child(4n+1) 这种方法是实现隔几选一的效果，比如我们这里是隔三选一，如果你把 4 换成别的数字那就是另外的一种隔法了，比如这个实例

.demo li:nth-child(4n+1) {
    background: lime;
}

我们主要来看其计算出来的结果

n = 0 -> 4n+1 = 1 -> 选择了第一个 li

n = 1 -> 4n+1 = 5 -> 选择了第五个 li

n = 2 -> 4n+1 = 9 -> 选择了第九个 li

:nth-last-child()

:nth-last-child() 选择器和前面的 :nth-child() 很相似，不同在于该选择器从最后一个元素开始算。我们来看几个实例：

.demo li:nth-last-child(4) {
    background: lime;
}

上面代码表示选择倒数第四个列表项，效果如下：

其中 :nth-last-child(1) 和 :last-child 所起作用是一样的，都表示的是选择最后一个元素。

另外 :nth-last-child() 也可以像 :nth-child() 一样，使用表达式来选择特定元素，下面我们来看几个特殊的表达式所起的作用

:nth-last-child(2n) 这个表示的是从元素后面计算，选择的是偶数个数，从而反过来说就是选择元素的奇数，和前面的 :nth-child(2n+1)、:nth-child(2n-1)、:nth-child(odd) 所起的作用是一样的。如：

.demo li:nth-last-child(2n) {
    background: lime;
}

.demo li:nth-last-child(even) {
    background: lime;
}

等于

.demo li:nth-child(2n+1) {
    background: lime;
}

.demo li:nth-child(2n-1) {
    background: lime;
}

.demo li:nth-child(odd) {
    background: lime;
}

请看效果：

:nth-last-child(2n-1) 这个选择器刚好跟上面的相反，从后面计算选择的是奇数，而从前面计算选择的就是偶数位了，这个前面的 :nth-child(2n) 之类是相同的效果，如：

.demo li:nth-last-child(2n+1) {
    background: lime;
}

.demo li:nth-last-child(2n-1) {
    background: lime;
}

.demo li:nth-last-child(odd) {
    background: lime;
}

等于：

.demo li:nth-child(2n) {
    background: lime;
}

.demo li:nth-child(even) {
    background: lime;
}

注意：

  • 在 iOS8 中 nth-child 类选择器存在不生效的 bug，需使用以下的 nth-of-type 代替

:nth-of-type

:nth-of-type 类似 :nth-child，不同的是他只计算选择器中指定的那个元素，这个选择器主要对用来定位元素中包含了好多不同类型的元素是很有用处，比如，div.demo 下有 p 元素，li 元素，img 元素等，但只需要选择 p 元素，并让他每隔一个 p 元素就有不同的样式，那我们就可以简单的写成：

.demo p:nth-of-type(even) {
    background-color: lime;
}

其实这种使用和 :nth-child 使用是一样的，也可以使用 :nth-child 的那些表达式和使用方法，唯一不同的是这种指定了元素的类型。

同样在 IE6-8 和FF3 以下浏览器不支持这种选择器

:nth-last-of-type

与前面的 :nth-last-child 一样使用，只是指定了元素的类型。

同样在 IE6-8 和FF3 以下浏览器不支持这种选择器

:first-of-type 和 :last-of-type

:first-of-type 和 :last-of-type 这两个选择器就类似于 :first-child 和 :last-child，不同之处就是指定了元素的类型。

同样在 IE6-8 和FF3 以下浏览器不支持这种选择器

:only-child 和:only-of-type

:only-child 表示的是一个元素是它的父元素的唯一一个子元素。

<div class="post">
    <p>Lorem ipsum dolor sit amet, consectetur</p>
    <p>Lorem ipsum dolor sit amet, consectetur</p>
</div>

<div class="post">
    <p>Lorem ipsum dolor sit amet, consectetur</p>
</div>

.demo .post p {
    background: lime;
}

初步效果

如果需要在 .post 只有一个 p 元素的时候，改变这个 p 的样式，那么我们就可以使用 :only-child，如：

.demo .post p {
    background: lime;
}

.demo .post p:only-child {
    background: red;
}

此时只有 .post 只有一个子元素 p 时，那么他的背景色将会改变,如图所示：

:only-of-type 是表示一个元素他有很多个子元素，而其中只有一个子元素是唯一的，那么我们使用这种选择方法就可以选择中这个唯一的子元素，比如说

<div class="post">
    <div>Lorem ipsum dolor sit amet, consectetur</div>
    <p>Lorem ipsum dolor sit amet, consectetur</p>
    <div>Lorem ipsum dolor sit amet, consectetur</div>
</div>

如果我们想只选择中上面中的 p 元素，我们就可以这样写，

p:only-of-type{
    background-color: red;
}

IE6-8 浏览器不支持 :only-child 选择器；IE6-8 和 FF3 以下浏览器不支持 :only-of-type 选择器。

:empty

:empty 是用来选择没有任何内容的元素。

假如有三个段落，其中一个段落是空的，需要不显示这个段落，可以这样来写：

p:empty {
    display: none;
}

IE6-8 浏览器不支持 :empty 选择器

否定选择器 :not

:not() 可以让你定位不匹配该选择器的元素。

假如你想对 form 中所有 input 加边框，但又不想密码输入框也起变化，此时就可以使用 :not 实现

input:not([type="password"]) {
    border: 1px solid red;
}

IE6-8 浏览器不支持 :not() 选择器

伪元素

CSS 中的伪元素主要包含 :first-line、:first-letter、:before、:after。

在 CSS3 中，对伪元素进行了调整，在以前的基础上增加了一个 : 与伪类进行区分。

因此在 CSS3 中包含的伪元素为 ::first-letter、::first-line、::before、::after。另外还增加了一个 ::selection。

目前由于为了兼容旧的浏览器，建议暂时沿用单个 : 的写法。

那么我们简单了解一下他们的作用。

::first-line 选择元素的第一行，比如说改变每个段落的第一行文本的样式，我们就可以使用这个

p::first-line {
    font-weight: bold;
}

::first-letter 选择文本块的第一个字母，除非在同一行里面包含一些其它元素，不过这个主要运用于段落排版上多，比如首字下沉

p::first-letter {
    font-size: 56px;
    float: left;
    margin-right: 3px;
}

::before 和 ::after 这两个主要用来给元素的前面或后面插入内容，这两个常与 content 配合使用，最常用的就是用于关闭浮动

.clearfix::before,
.clearfix::after {
    content: " ";
    display: table;
}
.clearfix::after {
    clear: both;
}

::selection 用来改变浏览网页选中内容的默认效果。
